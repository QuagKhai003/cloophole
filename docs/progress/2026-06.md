## 2026-06-23 ‚Äî fix: onefile self-spawn ate the window + _MEI warning (B11)
Branch `fix/onefile-self-spawn-env`. After B10, `cloophole open` still showed no
window ‚Äî but the decisive clue landed: a **python-parent** spawn of `cloophole.exe
_gui` with the *exact same* flags (CREATE_NO_WINDOW + DEVNULL) **opened the window**,
while the **frozen-parent** spawn (`cloophole open` ‚Üí itself) did not.

- **Root cause** ‚Äî PyInstaller onefile self-spawn. The frozen child inherited the
  parent's `_MEIPASS2`/`_PYI_*` env, so its bootloader attached to the PARENT's `_MEI`
  temp dir ‚Äî which the parent deletes on exit (the long-standing "Failed to remove
  temporary directory _MEI..." warning). The child then couldn't load bundled tcl/tk,
  so no Tk window appeared. Flags were never the real issue.
- **Fix** ‚Äî `runner._spawn` passes a child env stripped of `_MEI*`/`_PYI*` (new
  `_child_env()`), so the child extracts its own bundle like a clean launch. Fixes the
  no-window AND the `_MEI` warning in one go. +1 test. **29 ‚Üí 30.**
- **Note** ‚Äî the window itself was already proven good via foreground `_gui` (B7/B8/B10
  visible, no console, buttons fit); B11 was purely the launch/env path. User confirms
  on the rebuilt exe.

## 2026-06-23 ‚Äî fix: GUI opened hidden ‚Äî drop SW_HIDE (B10)
Branch `fix/gui-window-hidden-swhide`. User-reported: after the latest install,
`cloophole open` shows neither a window nor a terminal.

- **Root cause** ‚Äî the B8 fix added `STARTUPINFO` `STARTF_USESHOWWINDOW | SW_HIDE` to
  suppress the console. `wShowWindow=SW_HIDE` is the child's `nCmdShow`, which Tk
  applies to its first top-level window ‚Üí the GUI was created hidden. B8's source test
  only checked "process alive", not "window visible", so it slipped through.
- **Fix** ‚Äî `CREATE_NO_WINDOW` alone: suppresses the console for the console-subsystem
  exe without hiding the GUI window. Dropped the SW_HIDE STARTUPINFO. Test updated to
  assert no STARTF_USESHOWWINDOW.
- **Verification limit** ‚Äî only reproducible/confirmable on the frozen exe (source
  spawns windowless pythonw); per user's standing rule I do not launch the app here,
  so the user builds/installs and confirms. **29 tests.**
- **Process note** ‚Äî three GUI-launch bugs in a row (B7 crash, B8 console, B10 hidden)
  all live in `runner._spawn` flag handling; none are caught by the offline suite
  because they're OS-window behaviors. Real fix verification = user on the exe.

## 2026-06-23 ‚Äî fix: idle probe drained quota ‚Äî poll OFF by default (B9)
Branch `fix/poll-off-by-default`. User-reported: "drained my usage while doing
nothing." Root cause: `poll_enabled` defaulted **on**, so the daemon sent a real
`claude -p` probe every 30 min while WATCHING ‚Äî each costs quota, even when not
limited. A design flaw: it spends the resource the tool protects.

- **Mitigation** ‚Äî `config.DEFAULTS["poll_enabled"]` ‚Üí **False** (opt-in via `poll on`
  or the GUI checkbox). GUI checkbox now warns it spends usage. Test flipped
  (`test_poll_off_by_default`). DATA_MODEL + BUGS B9 logged.
- **Verified** ‚Äî `last_fired`/`last_poll` were null in state, ruling out a runaway
  resume loop; the probe is the only quota path, hence the default flip.
- **Planned real fix** ‚Äî replace polling with a Claude Code `StopFailure`/`rate_limit`
  hook: fires exactly when the limit hits, costs nothing, Golden-Rule clean (public
  hooks API, not internal reads). Hook `cwd` payload also fixes B6. Needs an ADR.
- **Tests:** 29 (unchanged count; one rewritten).

## 2026-06-22 ‚Äî fix: blank console window + clipped GUI buttons (B8)
Branch `fix/gui-window-console-and-size`. After B7 the window appeared, but with a
blank black terminal behind it, and its bottom buttons were cut off (no scroll).

- **Console window** ‚Äî `_spawn` passed `DETACHED_PROCESS | CREATE_NO_WINDOW` together.
  Win32 *ignores* CREATE_NO_WINDOW when DETACHED_PROCESS is set, so the console-
  subsystem exe got a visible blank console. (Latent until B7 let the GUI live.)
  **Fix:** CREATE_NO_WINDOW alone (no DETACHED) + STARTUPINFO SW_HIDE. The child still
  outlives the parent (separate process). +1 test asserting the flags.
- **Window size** ‚Äî 440x430 clipped the 6 action buttons. **Fix:** taller default +
  clamp geometry/minsize to `winfo_reqheight()` after layout (DPI/font safe).
- **Note** ‚Äî console fix only reproduces with the frozen console exe (source uses
  windowless pythonw); covered by unit test + Win32 docs, visually confirmed after CI
  rebuild. Window size verified live from source. **28 ‚Üí 29** tests.
- **Docs** ‚Äî BUGS B8 (resolved), STATUS, this entry.

## 2026-06-22 ‚Äî fix: `cloophole open` showed no window (B7)
Branch `fix/detached-gui-stdio`. User-reported: fresh install + `cloophole open`
started the watcher but **no window appeared** (only a benign PyInstaller `_MEI`
temp-cleanup warning).

- **Diagnosis** ‚Äî `gui.pid` was written but its process was dead, while the daemon
  lived. Foreground `_gui` ran fine (tkinter bundled OK); the detached spawn crashed.
  Reproduced: detached spawn **with** stdio‚Üífile/DEVNULL survives; **without** it dies.
- **Root cause** ‚Äî `runner._spawn` used `DETACHED_PROCESS` (no console) but left stdio
  inherited ‚Üí invalid stdout/stderr handles ‚Üí GUI dies on Tk's first write, right
  after writing `gui.pid`. Daemon survived only because it doesn't write to stdout early.
- **Fix** ‚Äî `_spawn` now passes `stdin/stdout/stderr=subprocess.DEVNULL`. Verified live
  from source (window survives 5s + clean stop). +1 regression test asserting the
  DEVNULL kwargs. **27 ‚Üí 28** tests.
- **Docs** ‚Äî BUGS B7 (resolved), STATUS (active task + "verify before trusting";
  corrected stale "pending merge `feat/gui-window`" ‚Äî it was already merged).
- **Note** ‚Äî released exe predates this; CI rebuilds on push to main, then reinstall.
